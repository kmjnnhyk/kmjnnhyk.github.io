Claude Code로 작업하다 보면 꽤 괜찮은 사고 과정이 나올 때가 있다. 가설 세우고, 검증하고, 틀리면 다시 돌아가고. 근데 세션 끝나면 그게 다 날아간다. 대화 히스토리야 남지만, 거기서 "내가 왜 이 판단을 했는지"를 다시 꺼내보기엔 너무 날것이다.

이걸 기술 블로그처럼 축적할 수 있으면 좋겠다는 생각이 들었다.

## 자동 저장은 안 되나?

처음엔 hook으로 세션이 끝날 때 자동 저장하는 걸 생각했는데, 확인해보니 hook이 받을 수 있는 데이터에 대화 내용이 없었다. 결국 명시적으로 "지금 기록해"라고 트리거하는 방식으로 갔다. 어차피 스킬로 만들면 대화 컨텍스트가 이미 메모리에 있으니까 깔끔하다.

## 기록과 발행을 분리

작업 세션 중간에 글을 다듬고 있으면 본래 작업이 끊긴다. 그래서 스킬을 두 개로 나눴다.

**기록 스킬**은 글로벌로 설치해서 어떤 프로젝트에서든 쓸 수 있게 했다. 세션에서 다룬 문제, 사고 과정, 판단 근거를 구조화해서 원재료로 쌓아둔다.

**발행 스킬**은 블로그 레포에서만 동작한다. 쌓인 히스토리를 블로그 글로 가공하고, 보안 필터를 거쳐서 GitHub Pages로 배포한다. 회사 코드나 기획이 그대로 나가면 안 되니까 실제 코드는 개념 수준으로, 내부 경로나 서비스명은 일반화한다.

블로그 프레임워크는 Jekyll로 갔다. GitHub Pages에 기본 내장이라 마크다운 push만 하면 배포 끝이니까.

## 남은 것

지금은 파이프라인의 뼈대만 잡은 상태다. 실제로 몇 번 써보면서 히스토리 포맷이나 문체 일관성을 다듬어야 할 것 같다. 일단은 세션이 휘발되지 않는다는 것만으로도 충분히 만족스럽다.
