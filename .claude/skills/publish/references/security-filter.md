# 보안 필터 규칙

publish 시 블로그 글에 포함하면 안 되는 내용.

## 제외 대상

### 코드
- 실제 코드 스니펫 포함 금지
- 개념/접근법만 서술, 구체적 구현은 의사 코드나 일반화된 예시로 대체
- 예시가 필요하면 공개 라이브러리 공식 문서 수준으로 일반화

### 기획/비즈니스
- 회사/제품 내부 기획 유출 금지
- 비즈니스 로직의 "왜"는 쓰되 구체적 기획 내용(기능 스펙, 유저 플로우 상세)은 제외
- 미출시 기능이나 로드맵 언급 금지

### 인프라
- API 엔드포인트 경로 제외
- 내부 서비스명, 마이크로서비스 구조 제외
- DB 스키마, 테이블 구조 제외
- 환경 변수 이름, 시크릿 키 제외
- 사내 도구/시스템 이름 제외

### 파일 경로
- 프로젝트 내부 파일 경로는 일반화하여 표현
- O: "설정 파일에서 transport 옵션을 추가했다"
- X: "`src/config/sentry.ts`의 `Sentry.init({...})`에 추가"

## 적용 방법

1. 히스토리 파일의 내용을 블로그 글로 변환할 때 위 규칙을 적용
2. 변환 후 초안을 사용자에게 보여줄 때, 보안 필터가 적용되었음을 알림
3. 사용자가 최종 검수 후 발행
